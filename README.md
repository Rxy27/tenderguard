- 遇到的问题：f-string 解决了什么问题？函数相比重复写代码有什么好处？两个概念不熟悉。

  f-string 主要解决了“把变量方便地放进字符串”的问题，让字符串拼接更直观。

  函数的主要好处是：
  减少重复：相同逻辑只写一次；
  便于修改：规则变化时通常只改函数内部；
  降低错误率：避免复制代码时漏改变量；
  提高可读性：函数名能说明代码在做什么；
  方便复用和测试：可以用不同参数反复调用；

## 核心项目：招标条款风险检测器

`tender_risk_checker.py` 可以接收项目名称和招标条款，根据预设规则检测风险关键词。

目前支持检测的关键词：

- 最低价中标
- 无限责任
- 不接受联合体

每条规则包含以下类似信息：

```python
{
    "keyword": "最低价中标",
    "level": "低风险",
    "reason": "可能导致供应商过度压价，影响项目质量"
}
```

程序通过下面的函数完成检测：check_tender_risks(text, rules)
