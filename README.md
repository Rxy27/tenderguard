## 概念总结

### 为什么使用 with open？

`with open(...)` 可以在代码块结束后自动关闭文件。即使读写文件时出现异常，文件也能正确关闭，可以避免忘记调用 `close()`。这部分目前理解得比较好。

### try / except 解决什么问题？

`try / except` 用来捕获程序运行时可能出现的异常。例如文件不存在时捕获 `FileNotFoundError`，JSON 格式错误时捕获 `json.JSONDecodeError`，然后输出友好提示，不让程序直接出现一整屏报错。这部分已经基本掌握。

### JSON 和 Python 字典有什么区别？

JSON 是一种用于保存和交换数据的文本格式，Python 字典是程序运行时内存中的数据结构。JSON 中的对象加载到 Python 后通常会变成字典，但两者的语法和支持的数据类型并不完全相同。这个概念目前还不够熟练，需要继续练习 JSON 和字典之间的转换。

### json.load 和 json.dump 分别做什么？

`json.load(file)` 从已打开的 JSON 文件中读取数据，并转换成 Python 对象。`json.dump(data, file)` 将 Python 对象写入 JSON 文件。这里容易混淆“路径字符串”、“文件对象”和“要写入的数据”，是目前需要重点复习的内容。

## 遇到的问题

1. 一开始把 JSON 文件路径直接传给 `json.load()`，出现 `'str' object has no attribute 'read'`。原因是 `json.load()` 需要的是已打开的文件对象，不是路径字符串。
2. 曾经直接在文件对象中匹配关键词。正确方法是先使用 `read()` 获得文本字符串，再用 `keyword in tender_text` 进行匹配。
3. 风险等级函数最初只使用 `print()`，导致后面无法把等级保存到报告中。修正后函数使用 `return` 返回风险等级。
4. 最初没有正确构造报告字典，并且混淆了 `json.dump()` 的参数顺序。正确写法是 `json.dump(report, file, ensure_ascii=False, indent=2)`。
5. `rule_loader.py` 目前可以打印读取结果和异常提示，但还没有按注释要求返回规则列表或空列表，后续需要补充 `return rules` 和 `return []`。
